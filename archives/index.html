Key points about AWS WAF

2019-06-16
  • AWS

1. Operational Excellence

  • Perform operations as code
  • Document everything
  • Push small changes instead of big
  • Refine operating procedures often
  • Anticipate failure
  • Learn from the operational failures

Areas

  • Prepare
  • Operate
  • Evolve

2. Security

  • Have a Strong Identity Foundation
  • Enable Traceability
  • Implement Security at All Layers
  • Secure the Data
  • Automate for Security
  • Plan for Security Events

Best Practices

  • Use IAM
  • Use Detective Controls (Config, Config rule, CloudTrail, CloudWatch, VPC flow logs, Inspector)
  • Use Infrastructure Protection
  • Use Data Protection
    • In transit (HTTPS, SSL/TLS, VPN/IPsec, SSH)
    • At rest (Encryption)
  • Use Incident Response (SQLi, XSS, DDoS)

3. Performance Efficiency

Design principles

  • Go global in a few clicks
  • Leverage new technologies such as serverless
  • Consume advanced technology
  • Leverage multiple technologies

Step 1: Selection

  • Compute (instances, containers, functions)
  • Storage
  • Network
  • Database

Step 2: Review

  • Define your infrastructure as code (CloudFormaiton)
  • Use CI/CD
  • Run benchmarks and do performance tests regularly

Step 3: Monitoring

4. Reliability

在IDEA上开发Servlet并部署到Tomcat

2018-10-23
  • Note
  • JetBrains
  1. New Project -> Java Enterprise -> 勾选Web Application -> Next。
  2. 项目创建好了之后，将通用lib文件复制到 web/WEB-INF/lib。
  3. 选择File -> Project Structure -> Libraries，将刚才的lib文件夹添加进去。
  4. 继续选择Artifacts，同样将lib文件夹添加进去。在这里可以直接点选右下角的fix快速更改。其他的保持不变，选择OK应用更改。
  5. 右上角选择服务器 -> Edit Configuration... -> Deployment，修改合适的Application context，其他的保持默认就行。
  6. 最后，运行项目或者debug进行测试。
